# 撰写第一篇文章

## 这一章你会学到什么

这一篇会把“会写 Markdown”推进到“真的能发出一篇文章”。

学完后，你应该能理解：

1. 什么是 front-matter
2. 一篇 Hexo 文章的头部配置怎么写
3. 正式文章和草稿有什么区别
4. 怎样从新建文章一路写到本地预览

如果说上一章是在学“写字规则”，这一章就是在学“发文流程”。

---

## 一篇 Hexo 文章通常长什么样

在 Hexo 里，一篇文章通常由两部分组成：

1. 头部配置
2. 正文内容

示意如下：

```md
---
title: 我的第一篇文章
date: 2026-04-06 20:00:00
tags:
  - Hexo
  - 入门
categories:
  - 建站学习
---

# 这里开始才是正文

这是我的第一篇文章内容。
```

中间那一段被 `---` 包起来的区域，就是 **front-matter**。

---

## 什么是 front-matter

front-matter 可以理解成“文章说明卡”。

它不是给读者直接看的正文，而是给博客系统看的配置。  
Hexo 会根据这些信息决定：

- 文章标题是什么
- 发布时间是什么
- 属于哪些标签
- 属于哪个分类
- 是否发布
- 链接怎么生成

所以你可以把它理解为：

> 正文是文章内容，front-matter 是文章元数据。

---

## 最常见的 front-matter 写法

这是一个非常适合初学者起步的版本：

```yaml
---
title: 我的第一篇文章
date: 2026-04-06 20:00:00
updated: 2026-04-06 20:00:00
tags:
  - Hexo
  - Markdown
categories:
  - 建站练习
---
```

### 这些字段分别是什么意思

`title`  
文章标题。通常会显示在文章页标题、列表页标题和浏览器标题等位置。

`date`  
文章创建时间。Hexo 会用它来生成归档、排序等内容。

`updated`  
文章更新时间。适合在你后续修改文章时同步更新。

`tags`  
标签。更像关键词，一篇文章可以同时有多个标签。

`categories`  
分类。更像栏目或归属路径，帮助你整理内容结构。

---

## 新手还会用到哪些字段

除了最基础的几个字段，你还可能遇到这些：

```yaml
---
title: 我的第一篇文章
date: 2026-04-06 20:00:00
excerpt: 这是一篇关于 Hexo 入门的文章。
cover: /images/hello-cover.jpg
comments: true
lang: zh-CN
---
```

简单理解：

`excerpt`  
摘要。有些主题会在文章列表页显示这段简介。

`cover`  
封面图。是否生效，取决于你的主题是否支持。

`comments`  
是否开启评论功能。前提是你的主题和评论系统已经配置好。

`lang`  
文章语言。多语言站点时更常见。

你会发现一个规律：

> front-matter 里有些字段是 Hexo 常用字段，有些字段则和主题功能有关。

所以如果某个字段你写了却没效果，不一定是你写错，也可能是当前主题不支持。

---

## 怎么创建第一篇文章

Hexo 官方写作流程里，最常见的起点是新建文章。

```bash
hexo new "我的第一篇文章"
```

执行后，Hexo 一般会在 `source/_posts/` 下创建一个新的 Markdown 文件，并自动带上一份基础 front-matter。

你接下来要做的事情很简单：

1. 打开这个新生成的 `.md` 文件
2. 修改标题、标签、分类等头部信息
3. 在下面开始写正文

---

## 一篇最小可发布文章示例

```md
---
title: 我的第一篇文章
date: 2026-04-06 20:00:00
tags:
  - 练习
  - Hexo
categories:
  - 博客起步
---

## 为什么我要开始写博客

我想把学习过程记录下来，也想训练自己把知识讲清楚。

## 今天先完成三件事

1. 学会创建文章
2. 学会写 front-matter
3. 学会本地预览

> 先把第一篇发出来，比一直等“完美开局”更重要。
```

这篇虽然不长，但已经是一篇完整文章了。

## 什么是草稿

草稿，就是“还没准备正式发布的文章”。

它特别适合下面这些情况：

- 你先记一个选题
- 你先写大纲，正文以后再补
- 你内容写到一半，还不想出现在正式站点里

草稿的好处是：  
**你可以先开始写，而不是等准备完美再开工。**

---

## 在 Hexo 里怎么创建草稿

官方写作命令支持直接新建草稿：

```bash
hexo new draft "我想先记下这个选题"
```

这样创建出来的文件，一般会放在 `source/_drafts/` 目录里。

和正式文章相比，它最大的区别是：  
默认不会出现在最终生成的网站内容里。

---

## 草稿怎么预览

如果你希望在本地预览时把草稿也一起显示出来，可以使用：

```bash
hexo server --draft
```

这样你在本地浏览器里就能看到草稿效果，但它仍然不算正式发布。

这很适合你边写边看排版。

---

## 草稿怎么变成正式文章

当你觉得草稿已经写得差不多了，可以用发布命令把它转成正式文章：

```bash
hexo publish "我想先记下这个选题"
```

发布之后，它会从草稿区进入正式文章流程，后续你再生成和部署站点时，这篇文章就会被带上。

如果你更习惯显式写出类型，也常会看到带布局参数的写法。

---

## 一个推荐的新手写作流程

你可以先不用追求复杂工作流，按下面这套最稳：

### 方案 A：想到就直接写正式文章

适合已经确定会发的内容。

1. `hexo new "文章标题"`
2. 补 front-matter
3. 写正文
4. `hexo server` 本地预览
5. 满意后再生成和部署

### 方案 B：先写草稿再发布

适合你只是先存想法，或者内容还没完成。

1. `hexo new draft "选题名称"`
2. 在 `_drafts` 里慢慢写
3. `hexo server --draft` 预览
4. 准备好后再 `hexo publish "选题名称"`

如果你还在培养写作习惯，我更推荐从方案 B 开始。  
它能明显减少“还没想好就不敢动笔”的心理压力。

## 初学者最容易踩的坑

### 1. front-matter 格式写乱

front-matter 常见格式是 YAML，所以要特别注意：

- 冒号后面通常要留空格
- 缩进要统一
- 列表项前面要有 `-`
- 头尾的 `---` 不要漏

比如下面这样就更稳妥：

```yaml
tags:
  - Hexo
  - 学习记录
```

而不是随手把结构写乱。

### 2. 以为写了字段就一定生效

不是所有 front-matter 字段都会自动显示。  
有些字段需要主题支持，比如封面、摘要、置顶、目录等。

### 3. 草稿写了却找不到

很多时候不是丢了，而是你忘了用 `hexo server --draft` 来预览。

### 4. 把“开始写”拖成“准备写”

这其实是最常见的问题。  
你越早接受“先有草稿，再慢慢打磨”，写作阻力就会越小。

## 一个适合直接照用的文章模板

```md
---
title: 这里写文章标题
date: 2026-04-06 20:00:00
updated: 2026-04-06 20:00:00
tags:
  - 标签一
  - 标签二
categories:
  - 分类名称
---

## 这篇文章想解决什么问题

先用一两段话说明背景。

## 具体过程

1. 第一步
2. 第二步
3. 第三步

## 最后总结

写下你真正得到的结论，而不是只重复过程。
```

如果你不知道第一篇文章该怎么开头，这个模板已经足够你起步。

## 小练习

请你尝试完成下面两件事：

1. 写一篇名为“我的第一篇文章”的正式文章头部配置
2. 再写一个名为“以后想写的 3 个选题”的草稿

哪怕正文只写两三段，也没关系。  
这一步的重点不是写得多好，而是熟悉整套流程。

---

## 小结

写博客时，真正重要的不是“我会不会一个命令”，而是你能不能理解这套结构：

- front-matter 负责文章配置
- Markdown 负责正文内容
- 正式文章用于发布
- 草稿用于积累和打磨

只要你把这四件事分清楚，后面的写作效率会高很多。

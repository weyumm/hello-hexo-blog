# 前端与网页构成

## 这一章你会学到什么

学完这一篇，你应该能回答 3 个问题：

1. 前端到底是在做什么？
2. 一个网页为什么会同时出现文字、样式和交互？
3. HTML、CSS、JavaScript 分别负责哪一部分？

如果你现在还觉得“网页开发”听起来很复杂，先别急。可以把网页想成一间正在布置的小店：

- HTML 是店里的骨架和物品清单，决定“这里有什么”。
- CSS 是装修和陈列，决定“它看起来怎么样”。
- JavaScript 是店员的动作，决定“用户点击后会发生什么”。

理解这三个角色，前端入门就迈出了最关键的一步。

---

## 什么是前端

前端，简单说，就是用户在浏览器里**直接看到并操作的那一部分**。

比如你打开一个博客页面，能看到：

- 页面标题
- 正文文字
- 图片
- 导航栏
- 按钮
- 点击后的弹窗、切换、动画

这些都属于前端的工作范围。

和前端相对的，是后端。后端更像厨房，负责数据、账号、文章存储、接口等；前端更像餐厅门面，负责把内容用用户能理解、能操作的方式展示出来。

一句话记忆：

> 前端负责“展示与交互”，后端负责“数据与逻辑”。

---

## 网页是怎么来到你眼前的

当你在浏览器输入一个网址时，大致会发生下面这件事：

1. 浏览器向服务器请求网页文件。
2. 服务器把 HTML、CSS、JavaScript 等文件发回来。
3. 浏览器先读取 HTML，知道页面里有哪些元素。
4. 浏览器再读取 CSS，给这些元素穿上样式。
5. 浏览器再执行 JavaScript，让页面拥有交互能力。

所以你看到的网页，并不是一张“截图”，而是浏览器把多种文件拼装后渲染出来的结果。

---

## HTML：决定网页里有什么

HTML 的全名是 HyperText Markup Language，中文通常叫“超文本标记语言”。

它不是编程语言，更像是一套“标记规则”，用来告诉浏览器：

- 这里是一段标题
- 这里是一段正文
- 这里有一张图片
- 这里有一个按钮
- 这里有一个链接

来看一个最小例子：

```html
<h1>欢迎来到我的网页</h1>
<p>这是第一段介绍文字。</p>
<button>点我一下</button>
```

上面这段代码的意思很直白：

- `<h1>` 表示一级标题
- `<p>` 表示段落
- `<button>` 表示按钮

你可以把 HTML 理解为网页的“零件表”。

### 常见 HTML 元素

初学阶段最常见的几个标签有：

| 标签 | 作用 |
| --- | --- |
| `h1` ~ `h6` | 标题 |
| `p` | 段落 |
| `a` | 链接 |
| `img` | 图片 |
| `button` | 按钮 |
| `div` | 通用容器 |
| `ul` / `li` | 列表 |

其中 `div` 很常见，它本身没有特别强的语义，通常用来装其他内容，方便布局和分组。

---

## CSS：决定网页长什么样

CSS 的全名是 Cascading Style Sheets，中文一般叫“层叠样式表”。

如果说 HTML 负责“放东西”，那 CSS 就负责：

- 文字颜色
- 背景颜色
- 字体大小
- 边框
- 间距
- 排版布局
- 动画效果

比如下面这段 CSS：

```css
h1 {
  color: #2563eb;
  font-size: 32px;
}

button {
  background: #111827;
  color: white;
  border: none;
}
```

它的意思是：

- 让标题变成蓝色，字号更大
- 让按钮背景变深色，文字变白色

### HTML 和 CSS 怎么配合

HTML：

```html
<h1>我的博客</h1>
<button>开始阅读</button>
```

CSS：

```css
h1 {
  color: tomato;
}

button {
  background: black;
  color: white;
}
```

浏览器会把它们组合起来，于是你就能看到“有样式的标题”和“有设计感的按钮”。

所以：

> HTML 让网页有内容，CSS 让网页更好看。

---

## JavaScript：决定网页会不会动

JavaScript 是网页里的交互脚本语言。

它负责让网页从“静态展示”变成“可响应用户操作”的状态。比如：

- 点击按钮后弹出提示
- 点开菜单后展开导航
- 提交表单前做检查
- 自动切换轮播图
- 根据输入实时更新内容

看一个简单例子：

```html
<button onclick="changeText()">点我</button>
<p id="message">你好，我还没变化。</p>

<script>
  function changeText() {
    document.getElementById("message").textContent = "我被点击后改掉了。";
  }
</script>
```

这里发生了什么？

1. 页面里有一个按钮。
2. 用户点击按钮时，会执行 `changeText()`。
3. JavaScript 找到 `id` 为 `message` 的段落。
4. 把它的文字改掉。

于是，网页就有了“点击后变化”的能力。

---

## 一个完整网页通常由三层组成

为了更容易记忆，你可以用这个“三层结构”来理解网页：

### 1. 结构层：HTML

负责内容和语义。

示例：标题、段落、图片、按钮、列表。

### 2. 表现层：CSS

负责样式和布局。

示例：颜色、间距、圆角、阴影、弹性布局。

### 3. 行为层：JavaScript

负责交互和动态效果。

示例：点击切换、弹窗、表单验证、内容更新。

你以后写网页，基本都会同时和这三层打交道。

---

## 一个小例子：把三者放在一起

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8" />
  <title>我的第一个网页</title>
  <style>
    body {
      font-family: sans-serif;
      padding: 24px;
    }

    .card {
      background: #f3f4f6;
      padding: 16px;
      border-radius: 12px;
    }

    .active {
      color: #dc2626;
    }
  </style>
</head>
<body>
  <div class="card">
    <h1 id="title">欢迎来到前端世界</h1>
    <p>HTML 管结构，CSS 管样式，JavaScript 管交互。</p>
    <button onclick="highlightTitle()">试试看</button>
  </div>

  <script>
    function highlightTitle() {
      document.getElementById("title").classList.toggle("active");
    }
  </script>
</body>
</html>
```

你可以这样拆解它：

- HTML：创建标题、段落、按钮、卡片容器
- CSS：让卡片有背景、圆角、内边距
- JavaScript：点击按钮时切换标题颜色

这就是一个最基础但完整的网页案例。

## 初学者最容易混淆的点

### 1. HTML 不是用来“美化”的

很多人一开始会觉得“网页内容都写在 HTML 里，那颜色是不是也写这里？”  
答案是：**不推荐。**

HTML 更适合表达“这是什么”，而不是“它长什么样”。

### 2. CSS 不是用来“处理业务逻辑”的

CSS 可以做动画、悬停效果，但它不擅长复杂判断。  
一旦涉及“点击后计算、切换、请求数据”，通常就该交给 JavaScript。

### 3. JavaScript 不是只能做“特效”

很多人以为 JavaScript 就是做动画。其实它更重要的价值是：  
**让页面和用户产生互动。**

---

## 学到这里，你应该建立的直觉

看到一个网页时，你可以试着问自己：

- 这部分内容是用什么 HTML 元素搭出来的？
- 它的颜色、布局、字体是谁控制的？
- 用户点击后发生的变化是谁写的？

如果你能把一个页面拆成这三类问题，你已经在用前端开发者的视角看网页了。

## 小结

这一篇最重要的不是背定义，而是建立分工意识：

- HTML：内容与结构
- CSS：样式与布局
- JavaScript：行为与交互

把这三者分清，你后面学写网页时就不会总觉得“所有代码都混在一起”。

# Markdown 语法速查

## 这一章你会学到什么

学完这一篇，你应该能做到 3 件事：

1. 用 Markdown 写出结构清晰的文章。
2. 用表格整理信息，而不是把内容堆成一大段。
3. 在博客支持的前提下，写出行内公式和独立公式块。

如果你之前习惯在 Word 里排版，Markdown 会让你很快发现一件事：  
**写内容和调格式，原来可以分开。**

你不用一直点加粗、改字号、调缩进，而是直接用简单的符号表达“这是一段标题”“这是一段引用”“这里是一段代码”。

---

## 什么是 Markdown

Markdown 是一种轻量级标记语言。

它的特点是：

- 语法简单
- 可读性强
- 适合写博客、文档、笔记
- 和 Git、Hexo、GitHub 等工具配合很好

比如下面这段：

```md
# 我的第一篇笔记

今天开始学习 Hexo。

- 先学 Markdown
- 再学 front-matter
- 最后发布文章
```

即使你还没渲染它，也基本能看懂它想表达什么。

这就是 Markdown 的魅力：  
**源码本身也尽量像正常文字。**

## 最常用的基础排版

### 1. 标题

用 `#` 表示标题，`#` 越多，层级越低。

```md
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
```

通常一篇文章里：

- 一级标题用作文章标题
- 二级标题用来分大段
- 三级标题用来分小节

尽量不要一篇文章里层级跳来跳去，否则读者会觉得结构很乱。

---

### 2. 段落与换行

Markdown 里最自然的做法，就是直接写段落。

```md
这是第一段。

这是第二段。
```

中间空一行，通常就会被识别成两个段落。

如果你只是想在同一段里换行，可以根据渲染环境使用：

- 行尾加两个空格
- 或者使用 `<br>`

但在写博客时，优先建议用正常段落，而不是频繁手动换行。

---

### 3. 强调

```md
**这是加粗**
*这是斜体*
***这是加粗加斜体***
~~这是删除线~~
```

显示效果通常会是：

- **这是加粗**
- *这是斜体*
- ***这是加粗加斜体***
- ~~这是删除线~~

初学者常见问题不是“不会写”，而是“写太多”。  
强调是为了突出重点，不是为了把半篇文章都加粗。

---

### 4. 列表

无序列表：

```md
- 苹果
- 香蕉
- 橙子
```

有序列表：

```md
1. 安装 Hexo
2. 创建文章
3. 本地预览
4. 发布站点
```

如果你在写教程、清单、步骤说明，列表会比整段文字更容易读。

---

### 5. 引用

```md
> 这是一段引用内容。
> 可以用来放提醒、摘录或者总结。
```

很适合放：

- 关键结论
- 原话摘录
- 温馨提示

例如：

> Markdown 的价值，不是让你排版更花，而是让你写作更轻。

---

### 6. 行内代码与代码块

行内代码：

```md
请运行 `hexo server` 预览站点。
```

代码块：

````md
```bash
hexo new "我的第一篇文章"
hexo server
```
````

如果你在写技术文章，代码块几乎是必备的。  
建议给代码块标注语言，比如 `bash`、`js`、`html`、`yaml`，这样高亮会更清晰。

---

### 7. 链接与图片

链接：

```md
[Hexo 官网](https://hexo.io/)
```

图片：

```md
![一张示意图](./images/example.png)
```

如果你是写博客，图片路径一定要养成习惯，不然文章迁移时很容易丢图。

---

## 表格怎么写

表格很适合整理“对比类”信息。

```md
| 工具 | 作用 | 难度 |
| --- | --- | --- |
| Markdown | 写内容 | 低 |
| Hexo | 生成博客 | 中 |
| GitHub Pages | 托管静态网页 | 低 |
```

渲染后大致会像这样：

| 工具 | 作用 | 难度 |
| --- | --- | --- |
| Markdown | 写内容 | 低 |
| Hexo | 生成博客 | 中 |
| GitHub Pages | 托管静态网页 | 低 |

### 表格对齐

你还可以控制对齐方式：

```md
| 左对齐 | 居中 | 右对齐 |
| :--- | :---: | ---: |
| A | B | C |
```

说明：

- `:---` 表示左对齐
- `:---:` 表示居中
- `---:` 表示右对齐

## 数学公式怎么写

这一部分先记住一个前提：

> Markdown 本身并不天然保证所有平台都支持数学公式。

是否能正确显示，取决于你的博客环境、主题和公式渲染配置。  
如果你的站点启用了数学公式渲染，常见写法通常是下面两种。

### 1. 行内公式

把公式写在 `$...$` 中间：

```md
质能方程可以写成 $E = mc^2$。
```

显示时通常会变成：  
质能方程可以写成 $E = mc^2$。

### 2. 独立公式块

把公式写在 `$$...$$` 中间：

```md
$$
\frac{-b \pm \sqrt{b^2 - 4ac}}{2a}
$$
```

常见例子：

```md
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

### 3. 常见数学符号

```md
\frac{a}{b}      // 分数
\sqrt{x}         // 根号
x^2              // 上标
x_1              // 下标
\sum             // 求和
\int             // 积分
\alpha \beta     // 希腊字母
```

如果你目前只是写一般博客，不一定马上会用到公式；但如果你写学习笔记、数学、物理、机器学习相关内容，这部分会很有用。

## 一篇博客里最常用的 Markdown 组合

你以后最常写到的，通常不是“所有语法”，而是下面这套组合：

- 标题
- 段落
- 列表
- 引用
- 代码块
- 链接
- 图片
- 表格

换句话说，先把这几种写顺手，已经足够覆盖大多数博客场景。

## 一个完整的小例子

```md
# 我的第一周学习记录

今天我开始学习 Hexo 和 Markdown。

## 这周做了什么

- 安装了 Hexo
- 学会了创建文章
- 了解了 GitHub Pages

## 本周结论

> 先学会稳定输出，再追求复杂效果。

```

## 初学者常见问题

### 1. 总想把 Markdown 当排版软件用

Markdown 更适合“结构清晰、内容优先”的写作。  
如果你老是纠结字号、空格、视觉微调，反而容易卡住。

### 2. 语法记不住

很正常。  
你不用一开始全背下来，把最常用的几种写法边写边记就够了。

### 3. 表格写得很痛苦

这也正常。  
表格不是 Markdown 最舒适的部分，初期可以先写简单表格；复杂表格再考虑借助编辑器插件。

## 小练习

请你尝试自己写一段 Markdown，满足下面 5 个要求：

1. 有一个一级标题
2. 有一个二级标题
3. 有 3 条无序列表
4. 有一段引用
5. 有一个 2 列 2 行的表格

如果你能自己写出来，这一篇就已经掌握得差不多了。

---

## 小结

Markdown 最重要的不是“花样”，而是“高效表达结构”。

- 标题让文章有层次
- 列表让信息更清楚
- 表格让对比更直观
- 公式让专业内容更规范

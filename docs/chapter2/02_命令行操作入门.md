# 命令行操作入门

## 为什么要学命令行

搭建 Hexo 博客的过程中，你会频繁用到命令行来执行各种操作——安装工具、初始化项目、生成页面、推送代码。

不用担心，你并不需要成为命令行专家，只需掌握几个最常用的命令就够了。

---

## 打开命令行

不同操作系统打开方式略有不同：

| 系统 | 工具 | 打开方式 |
| --- | --- | --- |
| Windows | 命令提示符 / PowerShell | 按 `Win + R`，输入 `cmd` 或 `powershell` |
| Windows | Git Bash | 安装 Git 后右键菜单中可找到 |
| macOS | 终端 | `Spotlight` 搜索"终端" |
| Linux | Terminal | 系统应用菜单中找到 |

> [!TIP]
> 在 Hexo 操作中，推荐在 Windows 上使用 **Git Bash**，它的命令和 macOS/Linux 终端更统一，报错也更少。

---

## 最常用命令速查

### `pwd` — 查看当前所在目录

```bash
pwd
```

输出示例：`/c/Users/YourName/blog`

---

### `ls` — 列出当前目录下的文件

```bash
ls
```

Windows 命令提示符中对应的是 `dir`，但在 Git Bash 里可以直接用 `ls`。

---

### `cd` — 切换目录

```bash
cd 目录名          # 进入某个子目录
cd ..             # 返回上一级
cd ~              # 回到用户主目录
cd /d/blog        # 直接跳到指定路径（Git Bash）
```

**示例：**

```bash
cd Desktop        # 进入桌面
cd blog           # 进入 blog 文件夹
cd ..             # 退回到上一层
```

---

### `mkdir` — 创建新目录

```bash
mkdir blog
```

这会在当前位置创建一个名为 `blog` 的文件夹。

---

### 命令行小技巧

| 技巧 | 说明 |
| --- | --- |
| `↑` / `↓` 方向键 | 浏览历史命令，不用重复输入 |
| `Tab` 键 | 自动补全文件名或目录名 |
| `Ctrl + C` | 中断当前正在运行的命令 |
| 路径中有空格 | 用引号包裹，如 `cd "My Blog"` |

---

## 动手练习

打开 Git Bash，按顺序执行以下命令：

```bash
cd ~              # 回到用户目录
mkdir test-blog   # 创建一个测试文件夹
cd test-blog      # 进入该文件夹
pwd               # 确认当前路径
cd ..             # 回到上一层
```

如果每一步都顺利执行，说明你已经掌握了后续操作所需的基础命令行技能。

---

## 小结

| 命令 | 作用 |
| --- | --- |
| `pwd` | 查看当前路径 |
| `ls` | 列出文件列表 |
| `cd 目录` | 进入目录 |
| `cd ..` | 返回上一级 |
| `mkdir 名称` | 新建文件夹 |

掌握这几个命令，就足以应对整个 Hexo 搭建流程中的命令行操作了。

# 配置文件详解

## 这一节你会学到什么

1. `_config.yml` 是什么，为什么重要
2. 最常用的配置项含义与修改方法
3. 如何设置博客标题、作者、URL 和文章链接格式

---

## 1. `_config.yml` 是什么

`_config.yml` 是 Hexo 博客的**核心配置文件**，位于博客根目录下。

它使用 [YAML](https://yaml.org/) 格式，以 `键: 值` 的形式书写，控制着博客的几乎所有行为：标题、语言、主题、部署地址……

用 VS Code 或任意文本编辑器打开它：

```bash
# 用 VS Code 打开（需安装 VS Code）
code _config.yml
```

> [!CAUTION]
> YAML 对缩进非常敏感，修改时注意：
> - 用**空格**缩进，不要用 Tab
> - 冒号后面**必须有一个空格**，如 `title: 我的博客`

---

## 2. 站点基本信息

这部分决定博客的身份信息，也会显示在浏览器标签和页面上。

```yaml
# 站点信息
title: 我的技术博客          # 博客标题
subtitle: 记录学习与成长      # 副标题（部分主题显示）
description: 一个 Hexo 博客  # 博客描述（用于 SEO）
keywords: Hexo, 博客, 技术   # 关键词（用于 SEO）
author: 你的名字             # 作者名，会显示在页脚等位置
language: zh-CN             # 语言，中文填 zh-CN
timezone: Asia/Shanghai     # 时区
```

---

## 3. 网址设置

```yaml
# 网址
url: https://你的用户名.github.io
root: /
```

- **`url`**：你博客的完整访问地址。部署到 GitHub Pages 后填写你的 `用户名.github.io` 地址；暂时没有域名可先留空
- **`root`**：站点根路径，通常保持 `/` 不变

---

## 4. 文章永久链接格式

```yaml
# 文章链接格式
permalink: :year/:month/:day/:title/
```

这控制每篇文章的 URL 格式。例如：

| 配置值 | 生成链接示例 |
| --- | --- |
| `:year/:month/:day/:title/` | `/2024/01/15/my-first-post/` |
| `:title/` | `/my-first-post/`（更简洁） |
| `:category/:title/` | `/tech/my-first-post/` |

> [!TIP]
> 推荐使用 `:title/` 格式，链接更简洁，对 SEO 也更友好。但要注意文章文件名不要包含中文，避免链接出现乱码。

---

## 5. 目录设置

```yaml
# 目录
source_dir: source    # 源文件目录（一般不用改）
public_dir: public    # 生成文件的输出目录
```

通常保持默认即可。

---

## 6. 主题设置

```yaml
# 主题
theme: landscape
```

`landscape` 是 Hexo 默认主题。后续安装 Butterfly 等主题后，将这里改为对应主题名称即可切换。

---

## 7. 部署设置

```yaml
# 部署
deploy:
  type: git
  repo: git@github.com:你的用户名/你的用户名.github.io.git
  branch: main
```

这个配置在本章下一节（部署到 GitHub Pages）会用到，这里先了解结构：
- `type`：部署方式，使用 Git 部署
- `repo`：你的 GitHub 仓库 SSH 地址
- `branch`：推送到的目标分支（通常是 `main`）

---

## 修改后验证

修改完 `_config.yml` 后，重新生成并预览，确认效果符合预期：

```bash
hexo clean && hexo g && hexo s
```

---

## 小结

| 配置项 | 作用 |
| --- | --- |
| `title` / `author` | 博客标题和作者名 |
| `url` | 博客的公开访问地址 |
| `permalink` | 文章的 URL 格式 |
| `theme` | 当前使用的主题 |
| `deploy` | 部署目标配置 |

配置文件改好之后，你的博客就有了"自己的名字"，下一步可以开始部署到 GitHub Pages 了。

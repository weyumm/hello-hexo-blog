# 本地博客预览

## 这一节你会学到什么

1. 用 `hexo generate` 将 Markdown 文章编译为静态网页
2. 用 `hexo server` 启动本地预览服务器
3. 理解端口号的概念
4. 在浏览器里看到你的博客初始效果

---

## 1. 两条关键命令

### `hexo generate`（简写 `hexo g`）

这条命令会读取 `source/` 目录下的 Markdown 文件和主题模板，将它们"编译"成浏览器可以直接显示的 HTML、CSS、JavaScript 文件，输出到 `public/` 目录。

```bash
hexo generate
# 或简写
hexo g
```

### `hexo server`（简写 `hexo s`）

这条命令会启动一个本地 Web 服务器，让你的浏览器可以访问刚才生成的静态文件。

```bash
hexo server
# 或简写
hexo s
```

---

## 2. 一键生成并预览

实际使用中，通常将两条命令合并：

```bash
hexo g && hexo s
```

> [!TIP]
> **PowerShell 用户注意**：`&&` 在部分 Windows 环境下可能报错，推荐在 **Git Bash** 中执行此命令。

启动成功后，终端会显示：

```text
INFO  Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.
```

在终端中按住 `Ctrl` 并左键单击该链接，即可在浏览器中打开本地博客。

## 3. 什么是端口号？

访问地址 `http://localhost:4000/` 中：

- **`localhost`**：表示"本机"，等价于 `127.0.0.1`，只有你自己的电脑才能访问
- **`4000`**：端口号，可以理解为"门牌号"

一台电脑可以同时运行多个服务，端口号用来区分它们。Hexo 默认使用 `4000` 端口，如果这个端口被其他程序占用，你可以用以下命令指定其他端口：

```bash
hexo s -p 5000   # 改用 5000 端口
```

---

## 4. 看到默认博客页面

打开浏览器后，你会看到 Hexo 的默认主题（Landscape），以及一篇默认文章 *Hello World*。

庆祝一下！这说明你的本地博客已经正常运行了。

---

## 5. 修改后刷新预览

每次修改文章或配置后，重新执行以下命令可以看到最新效果：

```bash
hexo clean && hexo g && hexo s
```

- `hexo clean`：清除旧的生成文件（避免缓存导致效果不对）
- `hexo g`：重新生成
- `hexo s`：启动预览

> [!NOTE]
> 每次确认修改满意后，再考虑推送到 GitHub，养成"先本地验证"的好习惯。

---

## 小结

| 命令 | 作用 |
| --- | --- |
| `hexo g` | 将 Markdown 编译为静态文件 |
| `hexo s` | 启动本地预览服务器 |
| `hexo clean` | 清除已生成的静态文件 |
| `hexo s -p 5000` | 指定端口启动 |

访问 `http://localhost:4000` 就能看到你的博客！
